个人博客搭建笔记

发表于 2017-11-10 | 阅读次数:

常常惊艳于充满个性的博客，炫酷的主题，丰富的排版，高标识的域名。心生羡慕，决定搭建一个博客，充实无聊的学生生活。本博客的搭建受益于网络上多位高级博主无私的奉献，是他们的优质博客加快了我的进度，再次谢过，同样也希望该文章对那些志同道合的朋友提供一点帮助。

平台和工具介绍

系统：win10
工具：github+hexo
说明：Hexo是时下比较流行的博客框架，有比较丰富的主题和插件支持。Github为用户提供了GitHub Pages，空间免费稳定，用来搭建一个博客是非常合适的。

##步骤

  • 本地环境配置（git、nodejs、hexo）
  • 创建github仓库
  • ssh连接
  • 初始化博客及配置
  • 打开githubpage
  • 购买配置域名
  • 编辑上传博客

本地环境配置

本地安装nodejs和git，安装方法比较简单，除非特殊需求，选择推荐选项安装即可。安装好git后打开git运行环境，使用npm安装hexo。

npm install -g hexo-cli

可能需要一段时间，之后会提示安装成功。

创建github仓库

不用多说，如果没有github账号的话先注册一个账号吧， 拥有一个github账号后创建一个name.github.io的仓库（name为你的个性化昵称）。

ssh连接

创建完仓库后是配置SSH密钥，用于连接github认证。

ssh-keygen -t rsa -C "email address" -f .ssh/name

之后回车确认，可能会提示输入密码，输入即可。当然，如果提示你没有ssh文件，这时你需要创建一个.ssh文件夹。之后执行上个步骤就成功了。
此时在.ssh文件下会出现几个文件，复制文件id_rsa.puh中的内容并到github上完成ssh 配置(下面链接1中有详细的图示)。

初始化博客及配置

以science作为博客名，进行说明

之后测试是否本地建站成功。

hexo s

正常的话会出现Hexo已经在localhost:4000端口上运行的提示，但是这时浏览器运行发现没有出现官网提示出现的页面，这种情况下我们可以尝试改一下端口，首先安装服务器模块。

npm install hexo-server --save

启动服务器并更改端口为5000.

hexo server -p 5000

浏览器打开本地端口

http://localhost:5000/

配置文件_config.yml在本地用户目录science下，主要修改基本信息如title、author等，重点修改deploy选项，已完成线上的访问。

之后本地生成并部署到远端

注意：在deploy是可能出现错误，没有发现git文件，这是需要安装deployer-git。

hexo-deployer-git --save

打开github page

如果我们发现使用http://science.git.io打不开我们的博客，可能是githubpage没有开启远程访问，这时我们进入science仓库，选择setting，找到下方的Github Pages选择以当前仓库的主干开启pages，注意此时下方会有一个custom domain选项，后面的修改域名就是改动该处。

购买配置域名

自己在万网上买了一个三块钱的域名（我自己名字的域名都被注册过了，无奈富贵命呀），修改上面提到的custome domain为自己购买的域名，github会提示你修改成功，当是当你访问自己域名的时候并没有找到你要的内容。原因是域名是有了，你没有dns呀，鬼知道该域名指向哪，这时候你就要去万网的控制面板中完成域名到github Ip地址的解析。如何查看github的ip。
在command命令行下执行：

ping github.io

显示的Ip就是你要完成域名解析对应的ip。
着急的你发现还是不能访问（晚上十点了，该回寝睡觉了，明天再来捣鼓），深深的打击。
第二天早上访问一下，奇迹出现了，分析为域名解析生效需要一段时间。至此，博客搭建完毕，本地编写上传吧，当然要用到markdown编辑器呀。

markdown编辑器

推荐两款windows下的编辑器markpad和farbox, 相比于这两款，一劳永逸的话还是推荐vscode上添加markdown插件的方式了。
